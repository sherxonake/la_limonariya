Навоийдаги La Limonariya ресторани учун **тўлиқ янги POS тизим — Clopos'ни БУТУНЛАЙ АЛМАШТИРАДИ.** **Муаммо:** ҳозирги Clopos таннарх ва омборни билмайди — «фойда» 97% сохта (реалда ~27%), склад бузуқ. **Ечим:** ўз тизимимиз — **касса + меню + тўлов + чек** (Clopos қиладигани) **ПЛЮС** кирим → обвалка → реал таннарх → реал қолдиқ → реал фойда → **дастур ўзи тешик/аномалия топади + башорат қилади**.

## 1. ЛОЙИҲА: МУАММО + ЕЧИМ

**Ҳозир (Clopos):** кунлик ~19–21 млн савдо, лекин таннарх йўқ → «фойда» сохта (97%). Склад −232 млн (минус, кирим киритилмаган). Обвалка, ярим-тайёр, аномалия — умуман йўқ.

**Биз қурамиз:** гўшт **қаёққа кетишини** тўлиқ кузатадиган, **ўзи таҳлил қиладиган** (тарихдан норма ўрганиб, аномалия топадиган) тизим. Энг катта пул рычаги — **шашлик грамми** (назорат ~25–28 млн/ой тежайди).

**Калит фактлар:** кунлик савдо ~21 млн · соф фойда ~27% · зарарсизлик ~9 млн/кун · маош йўқ (атрибуция) · кун чегараси 06:00 · смена битта.

## 1.5. ⭐ ДИЗАЙН ПРИНЦИПЛАРИ (энг муҳим — Clopos жонли таҳлилидан)

Clopos'нинг **склад/финансы/производство** модуллари жонли текширувда **бўш/бузуқ** чиқди (склад −232М, операции 0, производство 0) — чунки улар мураккаб ва workflow'дан узилган, ходим уларни **ишлатмаган**. Бизнинг тизим бу тақдирга тушмаслиги учун **3 та қатъий қоида:**

**1. 🎯 СОДДАЛИК (Simplicity)**
Ҳар амал минимал босиш билан. Бозорчи телефонда 3 та босиб кирим қилади · менежер бир экранда обвалка вазнини ёзади · официант 2 та босиб заказ беради. **Мураккаб = ишлатилмайди.**

**2. 🔒 МАЖБУРИЙЛИК (Enforcement)**
Муҳим амаллар workflow'да **мажбурий**, ходим унута олмайди:
- Тикетсиз таом ЙЎҚ (кухня тизим тикетисиз пиширмайди)
- Кун **ёпилмайди** инвентаризациясиз
- Чек **ёпилмайди** тўловсиз/атрибуциясиз

**3. 🔗 WORKFLOW'ГА БОҒЛИҚ (Integrated)**
Алоҳида «склад юритиш» керак эмас — кирим/обвалка/списание **табиий оқимда ўзи** юради: бозорчи харид қилади → авто кирим · сотув бўлади → авто списание · мариновка → авто ярим-тайёр.

> ⚠️ Агар бу 3 қоида бузилса — тизим Clopos склади каби **−232М ахлатга** айланади. **Соддалик = лойиҳанинг ютуғи, функция кўплиги эмас.**

## 5. WORKFLOW — 6 ОҚИМ

### 5.1 Харид + Обвалка
```
БОЗОРЧИ бозордан харид (гўшт/сабзавот/кўмир) → дастурга КИРИМ:
   тур + кг/дона + нарх + етказувчи(ихтиёрий) + тўлов(нақд/қарз)
   📍 GPS + 🕐 вақт АВТО · 📷 расм(ихтиёрий) · 💬 изоҳ(ихтиёрий)
        ↓
2 ПОВАР обвалка → МЕНЕЖЕР тарози вазнини ёзади
        ↓ қисмлар: премиум(вағора/шашлик/корейка), шапок, думба, фарш, жигар, суяк, чарви
ВАЗН БАЛАНСИ ±5% (ошса 🚩) → АНОМАЛИЯ ТЕКШИРУВИ (тарихий норма) → ОМБОРГА КИРИМ
```

### 5.2 Мариновка / Ишлаб чиқариш (ШАШЛИКЧИ қилади)
```
ХОМ ГЎШТ(кг) → МАРИНОВКА(партия, 2-4 кунлик) → СИХЛАШ → ВИТРИНА(дона)
Фарш/Рулет/Вафли → заготовка билан → витринага
```
Хом-ашё омбордан чиқади, ярим-тайёр кирим бўлади (ишлаб чиқариш ҳужжати).

### 5.3 Сотув (Официант мобил POS)
```
ОФИЦИАНТ телефонда (PIN) → стол/зал танлайди → меню → сони+изоҳ → ЮБОРАДИ
   → станция принтерларига чоп (мангал/ошхона/салат/бар)  [ҳали списание ЙЎҚ]
        ↓ тайёр → официант элтади
КАССИР «Ёпиш» босади → 💳 ТЎЛОВ ЭКРАНИ очилади
        ↓
   ТЎЛОВ ТУРИ ТАНЛАНАДИ (МАЖБУРИЙ):
   [💵 Нақд] [💳 Пластик] [🏦 Хумо] [📲 Клик] [🧾 Қарз]
   • Кўп-тўлов: бир неча тур аралаш (мас. 200к нақд + 100к қарз)
   • Қарз танланса → мижоз бириктириш МАЖБУРИЙ
   • Сумма = чек жами бўлмагунча «Ёпиш» тугмаси ўчиқ
        ↓
   ⛔ Биронта тўлов тури танланмаса → ЧЕК ЁПИЛМАЙДИ
        ↓
   ✅ ЧЕК ЁПИЛАДИ → 🔻 списание + хизмат ҳақи + атрибуция + касса журналига (тур бўйича)
```
- **🔒 МАЖБУРИЙ:** чек тўлов турисиз ёпилмайди. Касса журналида ҳар тур алоҳида (Excel'дагидек: клик/хумо/пластик ажралади).

**💳 Тўлов турлари жадвали (СОЗЛАНАДИГАН — Clopos'дагидек, лекин кенгроқ):**
Эга ўзи тур қўшади/таҳрирлайди. Ҳар тур учун флаглар:
| Майдон | Изоҳ |
|---|---|
| Номи | Нақд · Пластик · Хумо · Клик · Payme · Қарз... |
| Ҳисоб (Счёт) | қайси ҳисобга тушади (Касса / Карта / Банк) |
| 🔀 Бўлиш мумкин | кўп-тўлов (multi-tender) — аралаш тўлов рухсат |
| 👤 Мижоз мажбурий | қарз учун — мижоз танламай ёпилмайди |
| Актив/ноактив | кўрсатилсинми |

> Clopos'да 5 тур (Наличными/Картой/Баланс клиента/Кешбэк/Click) ва **«Можно разделить» + «Клиент должен быть выбран»** флаглари бор — биз шуни **кенгайтирамиз** (Хумо/Пластик/Payme алоҳида, эга ўзи майда/умумий танлайди).
- **Хизмат ҳақи ҳар зал:** Асосий 10% · Катта 10% · Терраса 15% · Собой 0%
- Бекор қилинган буюртма → омборга тегмайди
- Таом ўзгартириш (чопдан кейин) → директор рухсати
- **МАЖБУРИЙ ҚОИДА:** тикетсиз таом ЙЎҚ (тизимсиз сотув тешигини ёпади)

### 5.4 Қарз

## 7. СИХ / ВИТРИНА — 3 БОСҚИЧЛИ ҚОЛДИҚ
```
ХОМ ГЎШТ(кг) → [мариновка] → МАРИНАД(кг) → [сихлаш] → ВИТРИНА(дона) → [сотув] → списание
```
- Меню таом ↔ витрина сих **1:1** (Марварид сих, Корейка сих... ҳар тури алоҳида дона)
- **«Ҳисобланган сих ↔ қўлда киритилган сих» солиштируви:** 10кг ÷ грамм = кутилган сон; менежер реал сонни киритади; фарқ катта бўлса 🚩 (грамм оширилди ёки гўшт камайди)
- FIFO + эски сих огоҳ (муддат кейин созланади) · кеч қолган сих эртага қолади
- Рулет/Вафли ҳам сихга урилади (Вафли 140г/сих, Рулет 150г/сих)

### 🏷️ ПАРТИЯ ЭТИКЕТКАСИ (FIFO + муддат — янги)
Шашликчи партия тайёрлаганда (маринад/сих/фарш/рулет/вафли) → принтер **этикетка** чиқаради:
- Тур (мас. «Кусковой қўй маринад»)
- 📅 Тайёрланган сана/вақт · ⏳ Муддат (мас. 4 кун — созланади)
- 🔢 Партия ID + миқдор · 👤 ким тайёрлади
- (QR — скан қилиб витринага/назоратга улаш)

Этикетка идиш/падносга ёпиштирилади →
- **FIFO:** эски сана аввал сотилади (этикетка кўриниб туради — адашмайди)
- **Муддат:** муддати яқинлашса/ўтса 🚩 (этикетка санасидан авто)
- **Витрина:** этикетка QR скан → витринага кирим боғланади
> Бу — 3 босқичли қолдиқ ва муддат назоратининг **физик уланиши** (идишда кўриниб туради).

### ⭐ P&L АЖРАТИШ МАНТИҒИ (реал фойда — энг муҳим!)
Excel ҳаммасини аралаш «расход» қилади → реал фойда йўқ. **Биз ҳар чиқимни тўғри буккетга соламиз:**
```
КИРИМ (сотув)
 − COGS (бозорлик → омборга → сотилгандагина таннарх)   [харажат ЭМАС!]
 − OPEX (иш ҳақи + коммунал + ижара)                     [реал харажат]
 = СОФ ФОЙДА (РЕАЛ)
 [Эга олди, қарз = фойдадан ТАШҚАРИ — фақат пул ҳаракати]
```
- **Бир амал — икки ёзув (қўш ёзув йўқ):** бозорчи харид → касса − + омбор + (программа ўзи)
- Кассадан чиқим турлари: **харид**(пул ҳаракати) / **харажат**(P&L) / **эга олди**(тақсимот)
- Категориялар (реал Excel'дан): иш ҳақи · бозорлик · коммунал · эга олди · тўлов турлари

## 8.7. ⚖️ ФИСКАЛ / МАРКИРОВКА — ИККИ ОЛАМ (муҳим архитектура!)

Лимонария **солиққа уланмаган** (таом — ички/кулранг), ЛЕКИН **ичимлик расмий** (через счёт + маркировка апарати — Ўзбекистон қонуни). Тизим **2 оламни** битта POS'да бирлаштиради:

| | 🍽️ ТАОМ | 🥤 ИЧИМЛИК |
|---|---|---|
| Солиқ | ❌ ички | ✅ расмий (маркировка) |
| Харид | бозорчи (нақд) | через счёт (банк) |
| **Сотиш** | қўлда қўшилади | **СКАН мажбурий** (DataMatrix) — қўлда **БЛОК** |
| Ҳисоб | реал таннарх/фойда | расмий приход/расход |

**Маркировка интеграцияси (тасдиқланди):**
- 🔌 Апарат **бизнинг POS билан УЛАНАДИ** (API/маълумот ўқиш)
- Ичимлик сотиш = кассир **сканлайди** → апарат расмий рўйхатдан ўтказади → POS ҳам ёзади (омбор −, сотув +)
- ⛔ **Қўлда ичимлик қўшиб/сотиб бўлмайди** (фақат скан) → расмий из узилмайди
- Расмий приход (через счёт) → сотилганда расход

> 🎯 Натижа: **таом = реал фойда (ички)** · **ичимлик = расмий (маркировка)**. Иккаласи битта POS'да, ҳар хил оқим. Бу — ичимлик омборини (1.5-босқич) **расмий** қилади.

## 8.8. 🏗️ POS ҚУРИШ ДЕТАЛЛАРИ (раунд 17)

**💳 Тўлов:** 1-босқич **қўлда** (кассир «тўланди» белгилайди) → кейин Click/Humo/Payme **авто** (API).
**📱 Қурилма:** аралаш Android + iOS + десктоп → **PWA** (веб-илова, ҳаммасида бир хил ишлайди — Next.js).
**🍔 Меню бошқаруви:** **директор** (таом/нарх/категория қўшиш-таҳрирлаш).
**🌐 Сервер:** Шерхон ака ҳал қилади (offline-first — локал/булут синхрон).

**🖨️ Принтерлар (бизда БОР — 5 кухня + 1 касса):**
- Заказ **қоғозда** чиқади (KDS экран ЭМАС)
- Станция бўйича: мангал · ошхона · салат · балиқ · нон-чой + касса
- 2 хил чоп: ① **кухня тикети** (тайёрлаш) · ② **мижоз чеки** (ҳисоб)

**🔢 Чек/заказ рақами (ГИБРИД):**
- Ички: кетма-кет ID (ҳеч нолланмайди — audit/база)
- Кўрсатиладиган: **кунлик рақам** (ҳар кун 1 дан — «Стол 5, #042»)
- Асосан **ички назорат** (ошхона тикети + кассир кузатиши); мижоз чекида ҳам, лекин норасмий (таом фискал эмас)

**🔄 СМЕНА (X/Z ҳисобот):**
- **Очиш:** кассир PIN + бошланғич нақд (50к)
- **Ёпиш:** нақд саноқ → **Z-ҳисобот** (сотув: нақд/карта/клик/хумо/қарз алоҳида · чиқим · нечта чек · ўртача чек · **камомад**) → директор тасдиқ
- **X-ҳисобот:** оралиқ кўриш (ёпмасдан) · **Топшириш:** смена ёпиб-янгисини очиш
- Ҳозир битта смена (кун=смена), тизим кўпга тайёр

## 9. ВАЗИФА НАЗОРАТИ + ШТРАФ
Битта тизим, кўп вазифа. Ҳар вазифа: **чек-лист + дедлайн + расм + админ тасдиғи + штраф**.
- **3 тур:** ① дедлайн (бир марта) · ② даврий/interval (ҳожатхона ҳар 3 соат) · ③ воқеа-триггер+таймер (чек ёпилгач 10 мин → стол йиғиштириш)
- 📸 Расм = **фақат жонли камера** (галереядан эски расм юклаб бўлмайди, timestamp)
- Штраф **per-task** созланади: зинапоясимон (1-марта 30к/2-марта 50к/3-марта 100к, ҳар ой нолланади) ёки фикс (мас. стол 40к)
- «Ходим штраф/интизом журнали» — директор кўради

**Вазифалар:** Серверовка(стол 11:00) · Техничка(жадвал 8-18) · Кўрачи/Шашликчи(мангал) · Повар(кухня очилиш) · Кун охири(менежер) · Официант(стол йиғиштириш).
