## 💡 ВАРИАНТЛАР — қандай программага кирамиз

### 🅰️ Касса/харажат — 3 вариант

**Вариант A — «Excel'дек» кунлик журнал (1:1)**
Худди Excel: кунлик саҳифа (бошланғич қолдиқ → кирим қаторлар → чиқим қаторлар → охирги қолдиқ), қўлда. Таниш, лекin ҳаммаси қўлда.

**Вариант B — Авто-касса + тез харажат (ақлли)**
- Кирим **АВТО** (POS сотувидан) · тўлов турлари (нақд/клик/хумо) авто-ажралади
- Харажат: «+ харажат» тугмаси → категория + сумма + изоҳ/расм
- Қолдиқ авто ҳисобланади → камомад авто

**Вариант C — ГИБРИД ⭐ (тавсия)**
Excel'нинг **таниш кўриниши** + программанинг **автомати**:
- Кунлик журнал кўриниши (Рустам акага таниш)
- Кирим = авто (POS) · тўлов турлари авто
- Харажат = тез киритиш (категория тугмалари: 💵иш ҳақи · 🛒бозорлик · 💧коммунал · 👑эга олди · 📦бошқа)
- **Кунлик иш ҳақи** = алоҳида экран (ходимни белгилаб, сумма) — Excel'да салатчи/повар/шашликчи ҳар куни тўланарди
- Қолдиқ авто → камомад авто → **реал P&L га уланади**

> 🎯 **Тавсия: C** — таниш, лекин қўл меҳнати кам, реал фойдага уланади.

### 🅱️ Қарз → бизнинг ҚАРЗ модули
Excel матрицасини ташлаймиз. Қарз = мижоз ledger (чек+изоҳ+SMS+рейтинг). **Ҳозирги қарздорлар қолдиғини** Excel'дан бошланғич маълумот сифатида **импорт** қиламиз.

### 🅲️ Посуда/жиҳоз инвентаризацияси → ЯНГИ модул (таклиф)
Идиш-товоқ (тарелка/вилка/стакан...) даврий саноғи:
- Рўйхат + дона + давр (ҳафта/ой)
- **Топшириқ-қабул** (ким топширди / ким қабул қилди) — Excel'дагидек
- **Синган/йўқолган** = фарқ → йўқотиш (реал пул!) → директор кўради
- Бу — **янги тешик** (идиш «оёқ» қилади) — биз ёпамиз

## 🔬 ЧУҚУР WORKFLOW — КАССА/ХАРАЖАТ

### 1. Кунлик касса оқими (ким, қачон, қандай)
```
КУН БОШИ: қолдиқ = кечаги охирги қолдиқ + размен 50к
   │
   ├─◄ СОТУВ (POS) ──────────► авто КИРИМ (нақд/клик/хумо/пластик авто ажралади)
   ├─◄ Қарз тўлови (қайтарди) ─► КИРИМ
   │
   ├─► БОЗОРЧИ харид ─────────► ЧИҚИМ  (⚠️ + омборга кирим — БИР амал, ИККИ таъсир)
   ├─► Кунлик иш ҳақи ────────► ЧИҚИМ (салатчи/повар/шашликчи/серверовка ҳар куни)
   ├─► Эга/оила олди ─────────► ЧИҚИМ (рустам/дадам/тогом)
   ├─► Коммунал ──────────────► ЧИҚИМ (газ/сув/кўмир/свет)
   │
КУН ОХИРИ: кутилган қолдиқ = боши + Σкирим − Σчиқим
   реал саналган ↔ кутилган → КАМОМАД → директор тасдиқ → сейф
```

### 2. ⭐ ЭНГ МУҲИМ ТОПИЛМА: Excel ҳаммасини «харажат» қилади — биз ТЎҒРИ ажратамиз
Excel'да бозорлик, иш ҳақи, эга олди — **ҳаммаси аралаш «расход»**. Шунинг учун реал фойдани кўрсатмайди. **Биз ҳар чиқимни тўғри «челак»ка соламиз:**

```
КИРИМ (сотув)
  − COGS (бозорлик → омборга → сотилганда таннарх)   ◄ Excel "бозорлик" шу ерга (харажат эмас!)
  − OPEX (иш ҳақи + коммунал + ижара)                ◄ Excel "ЗП/газ/сув" шу ерга (реал харажат)
  = СОФ ФОЙДА (реал!)
  [Эга олди, қарз = фойдадан ТАШҚАРИ — фақат пул ҳаракати]
```

> 🎯 Бу — «сохта фойда → реал фойда» нинг **касса даражасидаги** ечими. Бозорликни харажат эмас, **омбор→COGS** қиламиз (сотилгандагина таннарх бўлади). Эга олганини фойдадан айирмаймиз (бу пул олиш, харажат эмас). Шу билан P&L ҲАҚИҚИЙ чиқади.

### 3. Интеграция — ҳар ҳаракат қаердан келади (қўш ёзув йўқ)
| Ҳаракат | Касса таъсири | Бошқа таъсир |
|---|---|---|
| Сотув (POS) | + Кирим (авто) | Списание (омбор −) |
| Бозорчи харид | − Чиқим | Омборга кирим (COGS асоси) |
| Кунлик иш ҳақи | − Чиқим | OPEX (фойдага таъсир) |
| Эга олди | − Чиқим | Фойда тақсимоти (P&L'га эмас) |
| Коммунал | − Чиқим | OPEX |
| Қарз бериш/олиш | ± | Мижоз баланси (P&L'га эмас) |

> **БИР амал — ИККИ ёзув автомат:** бозорчи харидни киритса, ҳам касса камаяди, ҳам омбор тўлади. Рустам ака иккита жойга ёзмайди — программа ўзи қилади.

### 4. Ким нима қилади (роллар)
- **Кассир:** сотув кирими (авто), кичик чиқим, кун охири нақд саноқ
- **Менежер:** харажат киритади (иш ҳақи, коммунал, бозорлик тасдиғи)
- **Бозорчи:** харид → авто чиқим + омбор кирим
- **Директор:** эга олди, ҳаммасини тасдиқлайди, реал P&L кўради

### 5. Кунлик иш ҳақи — алоҳида нозиклик
Excel'да салатчи/повар/шашликчи/серверовка **ҳар куни** тўланади (291×, 181×, 121×...). Бу OPEX'нинг катта қисми. → Программада **«Кунлик иш ҳақи» экрани:** ходимни белгила → сумма → касса камаяди + OPEX'га ёзилади. (Бу олдинги «маош йўқ» билан зид эмас — табел/зарплата модули йўқ, лекин кунлик берилган пул OPEX сифатида ёзилади.)
