## 🔄 ТЎЛИҚ WORKFLOW (тасдиқланган оқим)

```
ГЎШТ КЕЛАДИ ──► ПОВАРЛАР ОБВАЛКА ──► ЭЛЕКТРОН ТАРОЗИ ──► ОМБОР (қисмлар)
(BOZOR, нақд)   (туша → қисмлар)     (вазн авто-киради)   (қўй/мол лахм, шашлик,
                                                            корейка, думба, суяк,
                                                            фарш, чарви)
       │
       ▼
МАРИНОВКА (партия) ──► СИХЛАШ ──► «маринадланган шашлик» (ярим-тайёр SKU, омборда туради)
       │
       ▼
СОТУВ: официант буюртма ──► мангал (чоп) ──► КАССИР пул олади ──► ЧЕК ЁПИЛАДИ
                                                                       │
                                                                       ▼
                                                          СПИСАНИЕ (гўшт шу ерда ечилади)
       │
       ▼
КУН ЁПИЛИШИ (06:00) ──► нақд → сейф ──► директор паneli (фойда, аномалия)
```

## 🔑 ТАСДИҚЛАНГАН ҚАРОРЛАР (A–J)

### A. Списание вақти = **ЧЕК ЁПИЛГАНДА** ✅
Гўшт/маҳсулот омбордан **кассир пулни олиб, чек ёпилганда** ечилади.
- → Бекор қилинган (тўланмаган) буюртма омборга **тегмайди** (списание бўлмайди).
- → Реал-вақт қолдиқ = чек ёпилишига боғлиқ.
- → Официант буюртма берганда ёки мангалга чоп этилганда — ҳали списание **йўқ** (фақат "тайёрланмоқда" ҳолати).

### B. Шашлик = **маринадлаб → сихланади** (ярим-тайёр) ✅
Шашлик **мариновкадан кейин сихланади**. Демак олдиндан партия қилиб тайёрланади.
→ **Модел (муҳим):** «маринадланган/сихланган шашлик» = **алоҳида ярим-тайёр SKU**, омборда қолдиқ сифатида туради. (PLAN буни тасдиқлайди: "маринадланган гўшт алоҳида қолдиқ".)

**Икки босқичли списание:**
1. **Мариновка (ишлаб чиқариш ҳужжати):** хом гўшт (лахм) омбордан **чиқади** → маринадланган шашлик ярим-тайёр **кирим** бўлади. (Ички ҳаракат, "сотув" эмас.)
2. **Сотув (чек ёпилганда):** маринадланган шашлик ярим-тайёри тех-карта бўйича **ечилади**.

> Шу модель A (чек ёпилганда списание) ва B (мариновка) ни бирлаштиради: сотув-списание чек ёпилганда, лекин хом гўшт мариновкада аллақачон ярим-тайёрга айланган. Кунлик инвентаризацияда омборчи **иккаласини** ҳам санайди (хом лахм + сихланган шашлик).

### C. Обвалка: **поварлар ўзи, гўшт келганда, электрон тарози** ✅
- Обвалкани **поварлар ўзи** қилади (Маруф эмас — поварлар).
- **Ҳар гўшт келганда** обвалка бўлади (кунига бир марта эмас).
- **Электрон тарози** қўйилади — ҳар бир қисм шундан ўтади, вазн **автоматик** дастурга тушади. Қўлда ёзиш йўқ.
- → Ҳар бир қисм (шашлик, корейка, думба, шапок, суяк, фарш, чарви) **алоҳида тортилади** — "лахм"га умумлаштириб ташланмайди. Вазн баланси аниқ ёпилади.

### D. Харид: тарози **ўзи маълумотни беради** ✅ (нарх — кичик очиқ жой)
Гўшт келади → поварлар обвалка қилади → тарози вазн маълумотини **ўзи** беради.
> ⚠️ **Кичик очиқ савол:** тарози **вазнни** беради, лекин туша **нархи** (BOZOR'да нақдга тўланган сумма) қўлда киритилиши керак. Буни ким киритади — повар, омборчи ёки директор? *(D-2 да аниқлаймиз.)*

### E. 2 омбор орасида ҳаракат — ❓ жавоб берилмади
*(Қайта сўраймиз: обвалка қисмлари қайси муздаткичга тушади, кундузи ошхонага ким кўчиради.)*

### F. Смена = **БИТТА** ✅
Кунига **битта смена**. → Касса размени (50к) кунига бир марта, бошланғич қолдиқ = 50к.
> ⚠️ Кичик очиқ: кунни (06:00) ким ёпади — директорми ёки кассирми? *(F-2.)*

### G. 🔴 СУЯК — **ПУЛ КЕЛТИРАДИ** (модель ўзгариши!)
Рустам ака: «**Суяк пул кетиради** — суякни мангал қўямиз, ўша вақт аниқ бўлади».
→ **Топилма:** PLAN'да суяк "бепул" (нархи гўштга тарқалади) деб олинганди. Аммо реалда **суяк мангалга қўйилиб сотилади** — даромад келтиради!

**Янги модель тавсияси:**
- Обвалка таннарх ҳисоби **ўзгармайди**: суяк таннархи 0 (премиум гўштга тарқалади). Шунда премиум кесимлар тўғри қимматлашади.
- Лекин суяк **сотиладиган SKU** сифатида қўшилади: ўз сотув нархи билан. Таннархи ~0 бўлгани учун суяк сотуви = **деярли соф фойда (бонус)**.
- "Ўша вақт аниқ бўлади" = суяк мангалга кетганда вазни/қиймати реаллашади.

> ⚠️ **Очиқ:** суяк қандай сотилади — менюда алоҳида таом (нарх)ми, ёки мижозга шунчаки бериладими? Нархи қанча? *(G-2.)*
> Чарви ва брак (мусор) — жавоб берилмади, кейин аниқлаймиз.

### H. Аралаш тўлов = **ҲА, бўлади** ✅
Битта чек **бўлиниши** мумкин (масалан 200к нақд + 100к қарз, ёки Click + нақд).
→ POS **кўп-тўловли** (multi-tender) бўлиши керак: битта чекда бир неча тўлов тури.

### I. Собой workflow = **официант + кассир** ✅
Собой ҳам официант орқали ўтади, кассир пул олади. Атрибуция: **официант + кассир** (зал билан бир хил).

### J. Хизмат ҳақи: **Асосий + Катта зал 10%, Собой 0%** ✅
Фақат зал (Асосий) ва Катта зал = **+10%**. Собой = **0%**.
→ Clopos'даги "Терраса 15%" — **нотўғри/эскирган**, бизда Терраса ҳам зал = 10% (ёки умуман йўқ). Барча ўтириладиган зал = 10%, собой = 0%.

## 🎯 WORKFLOW'ГА ТАЪСИРИ (Шерхон ака учун)

1. **Списание дви́жок:** сотув списаниеси **чек ёпилишига** улаймиз (order/print эмас). Бекор қилинган буюртма омборга тегмайди.
2. **Ярим-тайёр қатлами:** маринадланган шашлик алоҳида SKU. Мариновка = ишлаб чиқариш ҳужжати (хом лахм → ярим-тайёр). Кунлик инвентаризация иккаласини ҳам санайди.
3. **Композит таомлар (Рулет, Вафли):** булар сих эмас — партия рецепти билан тайёрланадиган таом. Тех-карта порция бирлигида ёзилади.
4. **Суяк = даромадли SKU:** обвалка таннарх модели ўзгармайди (суяк cost=0), лекин суяк сотуви даромад сифатида ёзилади (~соф фойда).
5. **Кўп-тўловли чек:** битта чекда бир неча тўлов тури (multi-tender) — POS буни қўллаб-қувватлаши шарт.
6. **Хизмат ҳақи:** зал/катта зал 10%, собой 0% (Терраса 15% — бекор).
